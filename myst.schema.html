Node Type Index¶

FlowContent¶

Any of Paragraph | Definition | Heading | ThematicBreak | Blockquote | List | HTML | Code | Comment | Target | Directive | Admonition | Container | Math | Table | FootnoteDefinition

ListContent¶

Only ListItem

PhrasingContent¶

Any of StaticPhrasingContent | Emphasis | Strong | Link | LinkReference | Subscript | Superscript | Underline | Abbreviation | CrossReference | FootnoteReference

StaticPhrasingContent¶

Any of Text | HTML | EmphasisStatic | StrongStatic | InlineCode | Break | Image | ImageReference | Role | SubscriptStatic | SuperscriptStatic | UnderlineStatic | InlineMath

BlockBreak¶

Top-level break in the myst document, breaking it into Blocks

  • type*: string (“blockBreak”) - See Node

  • meta: string - block break metadata; conventionally, a stringified JSON dictionary but may be any arbitrary string

  • position: object (Position) - See Node

  • data: object - See Node

Block¶

Top-level content blocks or cells the myst document, delimited by BlockBreaks

  • type*: string (“block”) - See Node

  • meta: string - block metadata from preceding break; conventionally, a stringified JSON dictionary but may be any arbitrary string

  • children: array (FlowContent | ListContent | PhrasingContent) - Top-level children of myst document

  • position: object (Position) - See Node

  • data: object - See Node

Role¶

Custom in-line behavior

  • type*: string (“mystRole”) - See Node

  • kind*: string

  • value: string - content of the directive

  • children: array (PhrasingContent) - parsed role content

  • position: object (Position) - See Node

  • data: object - See Node

Directive¶

Content block with predefined behavior

  • type*: string (“mystDirective”) - See Node

  • kind*: string

  • args: string

  • options: object

  • value: string - body of the directive, excluding options

  • children: array (FlowContent | PhrasingContent) - parsed directive content

  • position: object (Position) - See Node

  • data: object - See Node

Target¶

Target node - provides identifier/label for the following node

  • type*: string (“target”) - See Node

  • identifier*: string - See OptionalAssociation

  • label: string - See OptionalAssociation

  • position: object (Position) - See Node

  • data: object - See Node

CrossReference¶

In-line reference to an associated node

  • type*: string (“crossReference”) - See Node

  • kind: string (“eq” | “numref” | “ref”)

  • children: array (StaticPhrasingContent)

  • identifier*: string - See OptionalAssociation

  • label: string - See OptionalAssociation

  • position: object (Position) - See Node

  • data: object - See Node

Abbreviation¶

Abbreviation node described by title

  • type*: string (“abbreviation”) - See Node

  • children*: array (StaticPhrasingContent) - abbreviated value - See Parent

  • title: string - advisory information for the abbreviation

  • position: object (Position) - See Node

  • data: object - See Node

Admonition¶

Admonition node for drawing attention to text, separate from the neighboring content

  • type*: string (“admonition”) - See Node

  • kind: string (“attention” | “caution” | “danger” | “error” | “hint” | “important” | “note” | “seealso” | “tip” | “warning”) - kind of admonition, to determine styling

  • class: string - admonition class info to override kind

  • children: array (AdmonitionTitle | FlowContent)

  • position: object (Position) - See Node

  • data: object - See Node

AdmonitionTitle¶

Custom title for admonition, replaces kind as title

  • type*: string (“admonitionTitle”) - See Node

  • children*: array (PhrasingContent) - See Parent

  • position: object (Position) - See Node

  • data: object - See Node

Container¶

Top-level container node to provide association and numbering to child content

  • type*: string (“container”) - See Node

  • kind*: string (“figure” | “table”) - kind of container contents

  • class: string - any custom class information

  • enumerated: boolean - count this container for numbering based on kind, e.g. Figure 1a

  • enumerator: string - resolved enumerated value for this container

  • children*: array (Caption | Legend | Image | Table) - See Parent

  • identifier: string - See OptionalAssociation

  • label: string - See OptionalAssociation

  • position: object (Position) - See Node

  • data: object - See Node

Caption¶

Caption for container content

  • type*: string (“caption”) - See Node

  • children*: array (FlowContent) - See Parent

  • position: object (Position) - See Node

  • data: object - See Node

Legend¶

Legend for container content

  • type*: string (“legend”) - See Node

  • children*: array (FlowContent) - See Parent

  • position: object (Position) - See Node

  • data: object - See Node

FootnoteReference¶

Inilne reference to footnote

  • type*: string (“footnoteReference”) - See Node

  • identifier*: string - See OptionalAssociation

  • label: string - See OptionalAssociation

  • position: object (Position) - See Node

  • data: object - See Node

FootnoteDefinition¶

Rich footnote content associated with footnote reference

  • type*: string (“footnoteDefinition”) - See Node

  • children*: array (FlowContent) - See Parent

  • identifier*: string - See OptionalAssociation

  • label: string - See OptionalAssociation

  • position: object (Position) - See Node

  • data: object - See Node

Math¶

Math node for presenting numbered equations

  • type*: string (“math”) - See Node

  • enumerated: boolean - count this math block for numbering based on kind, e.g. See equation (1a)

  • enumerator: string - resolved enumerated value for this math block

  • identifier: string - See OptionalAssociation

  • label: string - See OptionalAssociation

  • value*: string - See Literal

  • position: object (Position) - See Node

  • data: object - See Node

InlineMath¶

Fragment of math, similar to InlineCode, using role {math}

  • type*: string (“inlineMath”) - See Node

  • value*: string - See Literal

  • position: object (Position) - See Node

  • data: object - See Node

Table¶

Two-dimensional table data

  • type*: string (“table”) - See Node

  • align: string (“left” | “center” | “right”)

  • children*: array (TableRow) - See Parent

  • position: object (Position) - See Node

  • data: object - See Node

TableRow¶

One row of table containing cells

  • type*: string (“tableRow”) - See Node

  • children*: array (TableCell) - See Parent

  • position: object (Position) - See Node

  • data: object - See Node

TableCell¶

One cell of table

  • type*: string (“tableCell”) - See Node

  • header: boolean

  • align: string (“left” | “center” | “right”) - alignment of content within cell

  • children*: array (PhrasingContent) - See Parent

  • position: object (Position) - See Node

  • data: object - See Node

  • type*: string (“subscript”) - See Node

List¶

  • label: string - See OptionalAssociation

  • value*: string - See Literal

  • position: object (Position) - See Node

Resource¶

Reference to external resource

  • url*: string

  • title: string - advisory information, e.g. for a tooltip

OptionalAssociation¶

Internal relation from one node to another; not required by node

  • identifier: string - identifier that may match another node; value is unparsed and must be normalized such that whitespace is collapsed to single space, initial/final space is trimmed, and case is folded

  • label: string - node label; character escapes and references are parsed; may be normalized to a unique identifier

Association¶

Internal relation from one node to another

  • identifier*: string - See OptionalAssociation

  • label: string - node label; character escapes and references are parsed; may be normalized to a unique identifier - See OptionalAssociation

Alternative¶

Alternative description of image

  • alt: string - field describing the image

Reference¶

Marker associated to another node

  • referenceType*: string (“shortcut” | “collapsed” | “full”) - explicitness of the reference: shortcut - reference is implicit, identifier inferred; collapsed - refernce explicit, identifier inferred; full - reference explicit, identifier explicit

Node¶

Base unist node

  • type*: string - identifier for node variant

  • data: object - information associated by the ecosystem with the node; never specified by mdast

  • position: object (Position) - location of node in source file; must not be present for generated nodes

Literal¶

Basic node with required string value

  • value*: string

  • type*: string - identifier for node variant - See Node

  • data: object - information associated by the ecosystem with the node; never specified by mdast - See Node

  • position: object (Position) - location of node in source file; must not be present for generated nodes - See Node

Parent¶

Basic node with required node children

  • children*: array (Node)

  • type*: string - identifier for node variant - See Node

  • data: object - information associated by the ecosystem with the node; never specified by mdast - See Node

  • position: object (Position) - location of node in source file; must not be present for generated nodes - See Node

Point¶

One place in a source file

  • line*: integer - line in the source file, 1-indexed

  • column*: integer - column in the source file, 1-indexed

  • offset: integer - offset character in the source file, 0-indexed

Position¶

Location of a node in a source file

  • start*: object (Point) - place of first character of parsed source region

  • end*: object (Point) - place of first character after parsed source region, whether it exists or not

  • indent: array (integer) - start column at each index in the source region, for elements that span multiple lines

Root¶

Myst syntax tree built on existing mdast schemas

  • type*: string (“root”) - See Node

  • children*: array (Block | BlockBreak | FlowContent) - Top-level children of myst document - See Parent

  • position: object (Position) - See Node

  • data: object - See Node
